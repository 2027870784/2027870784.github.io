# 项目经历

## [从零构建 C++ 3D 软光栅渲染器](./soft-render3d/)

**项目类型：** 计算机图形学个人项目  
**技术栈：** C++、CMake、MVP 变换、重心坐标插值、Z-Buffer、Lambert 漫反射  
**项目角色：** 独立开发

### 项目简介

本项目是在不依赖 OpenGL、DirectX 等高级图形 API 的前提下，使用 C++ 从零实现的软件 3D 渲染器，完成从模型数据到像素图像输出的基础渲染管线。

### 核心功能

- 手写向量、矩阵与齐次坐标相关数学运算
- 实现 MVP 变换、透视除法和屏幕空间映射
- 基于重心坐标完成三角形光栅化
- 使用 Z-Buffer 处理深度测试和遮挡关系
- 实现基础 Lambert 漫反射光照
- 输出 `.ppm` 图像并使用 CMake 组织工程

### 项目亮点

- 从零实践模型到像素的完整软件渲染流程
- 排查黑屏、深度写入、法线变换和输出性能等典型图形学问题
- 通过模块拆分沉淀可继续扩展的渲染核心
- [查看项目详情](./soft-render3d/)

## [个人技术门户与自驱动简历系统](./tech-portal-resume/)

**项目类型：** 个人工程实践项目  
**技术栈：** Hexo、Butterfly、Markdown、GitHub Pages、GitHub Actions、Git、YAML  
**项目角色：** 独立开发

### 项目简介

本项目基于 Hexo 静态网站生成器构建个人技术门户，使用 Markdown 管理技术文章、项目经历和简历内容，并通过 GitHub Actions 自动部署到 GitHub Pages。

### 核心功能

- Markdown 技术文章管理
- 项目经历展示
- 在线简历页面
- 响应式页面适配
- GitHub Pages 自动部署
- AI 简历自动化方案探索

### 项目亮点

- 使用静态网站生成器完成从内容到网页的自动转换
- 将个人简历和项目经历沉淀为可持续维护的在线技术资产
- 完成从本地开发、内容编写、静态构建到线上部署的完整工程流程
- [查看项目详情](./tech-portal-resume/)

## [AI 辅助多方向简历生成系统](./ai-resume-system/)

**项目类型：** 个人技术门户扩展项目  
**技术栈：** Markdown、YAML Front Matter、Node.js、JSON、Hexo、AI Prompt  
**项目角色：** 方案设计与原型开发

### 项目简介

本项目围绕个人技术门户中的项目经历和简历模块展开，目标是把项目 Markdown 内容结构化，并探索通过 AI 或本地规则生成不同方向的在线简历页面。

### 核心功能

- 使用 front matter 标注项目方向、技术栈、角色和排序权重
- 设计项目 Markdown 到结构化 JSON 的转换流程
- 按 Web、C++、通用软件开发等方向筛选项目经历
- 规划 AI 内容提炼与本地规则降级策略
- 为多方向简历页面提供统一内容来源

### 项目亮点

- 将项目经历、博客复盘和简历展示统一到同一套内容源
- 避免多份简历重复维护，降低长期更新成本
- 保留本地降级策略，避免外部 AI API 影响站点构建稳定性
- [查看项目详情](./ai-resume-system/)

## [Giscus 评论系统接入与 AI 审核方案](./giscus-comment-review/)

**项目类型：** 静态站点互动能力扩展  
**技术栈：** Hexo、Butterfly、Giscus、GitHub Discussions、GitHub Actions、YAML  
**项目角色：** 独立设计与配置

### 项目简介

本项目为 Hexo 技术门户接入 Giscus 评论系统，使静态站点具备基于 GitHub Discussions 的互动能力，并规划后续通过 GitHub Actions 或 Webhook 实现 AI 异步审核。

### 核心功能

- 配置 Butterfly 主题的 Giscus 评论系统
- 使用 GitHub Discussions 存储评论数据
- 按页面路径映射评论讨论
- 适配中文界面、浅色主题和深色主题
- 分析静态站点评论审核边界并设计异步审核方案

### 项目亮点

- 在不引入自建后端的前提下增加站点互动能力
- 明确 Giscus、GitHub Discussions 和静态站点之间的职责边界
- 为后续 AI 评论审核预留可扩展方案
- [查看项目详情](./giscus-comment-review/)

- 使用 Markdown 统一维护内容源

### 项目亮点

- 让站点内容从零散文章升级为可展示的项目档案
- 通过项目详情页提高验收展示时的信息密度
- 使用多方向简历展示同一批项目在不同目标下的能力侧重点

# 技术门户内容体系与验收展示优化

## 项目概览

**项目类型：** 内容工程与展示体验优化  
**项目角色：** 内容结构设计与页面维护  
**技术栈：** Hexo、Markdown、Butterfly、Git、信息架构  
**项目定位：** 将个人技术门户整理成适合项目验收和长期维护的展示系统

本项目围绕个人技术门户的内容体系展开，重点不是新增复杂功能，而是让博客、项目经历、简历和关于页面形成清晰结构，使站点在项目验收时能够完整展示学习过程、技术实践和阶段性成果。

## 项目背景

技术门户刚搭建完成时，内容容易呈现为零散文章。对于日常记录来说这已经够用，但在项目验收场景中，访问者需要更快理解项目价值。

因此，本项目对内容结构进行整理：新增多篇技术博客、扩展项目经历详情页、增加多方向简历，并更新导航入口和展示动线。

## 主要工作

- 梳理现有 `source/` 目录结构，确认博客、项目和简历的维护方式。
- 新增围绕 Hexo、GitHub Actions、AI 简历生成、Giscus 评论系统的技术文章。
- 在 `source/projects/` 下新增项目详情页，使项目列表有更多可点击内容。
- 在 `source/resume/` 下新增 Web / 前端方向和通用软件开发方向简历。
- 更新项目经历入口页和简历入口页，保证新增内容可以被正常访问。
- 控制内容真实性，避免虚构公司经历、上线规模或未完成能力。

## 内容结构

优化后的内容结构更适合展示：

```text
博客文章
-> 记录项目过程、技术选型和复盘

项目经历
-> 结构化展示项目背景、实现、问题和成果

在线简历
-> 按不同方向提炼能力和项目要点

关于页面
-> 说明站点定位和长期维护目标
```

这四类内容相互补充，共同构成完整的个人技术资产展示。

## 展示动线设计

用于项目验收时，可以按照下面的顺序展示：

1. 首页：展示站点整体形态和文章列表。
2. 项目经历：展示项目列表和多个详情页。
3. 软光栅渲染器：展示 C++ 与图形学实践深度。
4. 技术门户项目：展示站点本身的工程完整性。
5. AI 简历系统：展示内容自动化和扩展设计。
6. 简历页：展示不同方向的能力组织结果。
7. 博客归档：展示持续记录和复盘能力。

这条动线覆盖了功能、内容、工程实践和展示成果。

## 项目亮点

- 将站点从普通博客扩展为博客、项目、简历一体化的技术门户。
- 通过结构化项目页提高验收展示时的信息密度。
- 使用多方向简历展示同一批项目在不同方向下的能力侧重点。
- 内容全部由 Markdown 管理，便于后续继续维护。
- 站点自身也成为项目经历的一部分，形成“作品展示作品”的闭环。

## 项目成果

- 新增多篇技术博客，覆盖部署、内容结构、AI 简历和评论系统等主题。
- 新增多个项目经历详情页，丰富项目入口展示内容。
- 新增 Web / 前端方向与通用软件开发方向简历。
- 完成项目入口页和简历入口页更新，形成更完整的验收展示路径。

## 个人收获

这个项目让我意识到，项目展示不是简单堆内容，而是需要组织信息。真正能服务验收的站点，应该让访问者快速看到项目背景、实现过程、技术亮点和最终成果。

内容结构整理完成后，这个技术门户不只是博客集合，也更像一个可以持续维护的个人项目档案。
